# 模拟渗透模型

## 来源

5月8日上午，群内讨论拐点，之前有过一个模型思维的课程，里面用了一个渗透模型来讨论「什么情况下火灾会蔓延，为什么伟大的科技总是在同一时间段突然冒出来」

## 他是怎么做的

课程中用了netlogo，一个模型软件，假设有一个10X10的方格，从一个边界到另一个边界「渗透」，意味着一个伟大的发明的完成（像水从地表渗下去）。

解释一下就是，每个发明的背后都需要许多别的技术的成熟（比如iPhone的出现就需要有稳定质量的电容触摸屏）。

给每一个格子有了特定概率去「渗透」，然后从一个边界开始，每一次往下一层格子渗透都用概率计算成功率。

从0开始慢慢提高概率，然后看从哪一个概率开始，「渗透」开始出现，意味着一条新产品的产业链开始形成，就是从0到1，就是iPhone的电池、触摸屏、芯片都能商用量产。

然后随着概率（技术成熟度）的提高，渗透的成功率越来越高，接下来就是从1到100的生产。

## 这个python如何模拟

size里假设了是个10X10的小格子，然后从一边单向传递至对边，从0开始依次累加概率，一旦有渗透成功，输出此时的「渗透概率」，默认模拟重复100次，最后输出的概率第一个是极小值，第二个是平均值。

## 为什么不是59%

课程上教的是59%，为什么模拟不出来？因为这个程序做的是单向传递，而netlogo不光能单向，还能横向。
